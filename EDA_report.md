### 데이터 구조

```
sprint_ai_project1_data/
├── train_images/          # 학습 이미지 (PNG, 평탄 구조)
├── test_images/           # 테스트 이미지 (1.png, 2.png, ...)
└── train_annotations/     # 어노테이션 (개별 JSON 파일)
    └── {약조합폴더}/
        └── {약폴더}/
            └── {이미지명}.json   # 이미지 1장 + 어노테이션 1개
```

- 어노테이션은 단일 `train.json`이 아닌 **이미지 × 약 종류별 개별 JSON** 파일로 구성
- 한 이미지에 여러 종류의 약이 등장하는 경우 동일 이미지에 대한 JSON이 약 수만큼 존재
- JSON 포맷은 COCO 형식 (images / annotations / categories)
- 각 JSON에는 촬영 조건 메타데이터(카메라 각도, 조명, 앞뒷면 등) 포함

## 2. 어노테이션 구조 상세

### JSON 파일 구성

각 JSON 파일은 COCO 포맷 기반이며 최상위 키는 다음과 같다.

```json
{
  "images":      [...],   // 이미지 정보 (1개)
  "annotations": [...],   // BBox 정보 (1개)
  "categories":  [...],   // 약 카테고리 (1개)
  "type": "instances"
}
```

---

### `images` 필드

이미지 정보 외에 약의 상세 메타데이터를 함께 포함한다.

| 키 | 예시 값 | 설명 |
|----|---------|------|
| `file_name` | `K-001900-..._70_000_200.png` | 이미지 파일명 |
| `width` / `height` | `976` / `1280` | 이미지 해상도 |
| `drug_N` | `K-001900` | 약 식별 코드 |
| `drug_S` | `정상알약` | 약 상태 |
| `drug_dir` | `앞면` / `뒷면` | 촬영 방향 |
| `camera_la` | `70`, `75`, `90` | 카메라 촬영 각도 (°) |
| `camera_lo` | `0` | 카메라 수평 각도 |
| `light_color` | `주백색` | 조명 색상 |
| `back_color` | `연회색 배경` | 배경 색상 |
| `size` | `200` | 촬영 크기 설정값 |
| `dl_name` | `보령부스파정 5mg` | 약 이름 (한글) |
| `dl_name_en` | `Buspar Tab. 5mg Boryung` | 약 이름 (영문) |
| `dl_material` | `부스피론염산염` | 주성분 |
| `dl_company` | `보령제약(주)` | 제조사 |
| `di_class_no` | `[01170]정신신경용제` | 약효 분류 |
| `di_etc_otc_code` | `전문의약품` | 전문/일반 구분 |
| `drug_shape` | `장방형` | 약 모양 |
| `color_class1` | `하양` | 약 색상 |
| `print_front` / `print_back` | `BSP` / `5` | 약 표면 각인 |
| `form_code_name` | `나정` | 제형 |
| `thick` | `2.5` | 두께 (mm) |
| `leng_long` / `leng_short` | `8` / `4.5` | 장축/단축 길이 (mm) |

---

### `annotations` 필드

| 키 | 예시 값 | 설명 |
|----|---------|------|
| `id` | `133` | 어노테이션 고유 ID |
| `image_id` | `34` | 대응하는 이미지 ID |
| `category_id` | `1900` | 카테고리 ID (약 K-번호 숫자부) |
| `bbox` | `[644, 845, 189, 190]` | BBox 좌표 `[x_min, y_min, width, height]` |
| `area` | `35910` | BBox 넓이 (px²) |
| `iscrowd` | `0` | 군집 여부 (항상 0) |
| `segmentation` | `[]` | 세그멘테이션 (미사용, 빈 배열) |
| `ignore` | `0` | 무시 여부 |

> BBox 포맷은 **COCO 표준** `[x_min, y_min, width, height]` 이며,
> YOLO 계열 학습 시 `[x_center, y_center, width, height]` (정규화)로 변환 필요.

---

### `categories` 필드

| 키 | 예시 값 | 설명 |
|----|---------|------|
| `id` | `1900` | 카테고리 ID |
| `name` | `보령부스파정 5mg` | 약 이름 (한글) |
| `supercategory` | `pill` | 상위 카테고리 (전체 동일) |

---

### 파일명 규칙

```
K-{약1}-{약2}-...-{약N}_{앞뒷}_{불명}_{앞뒷}_{불명}_{각도}_{수평각}_{크기}.png
예: K-001900-016548-019607-029451_0_2_0_2_70_000_200.png
```

- 파일명에 포함된 `K-XXXXXX` 코드들이 해당 이미지에 등장하는 **약 조합**을 나타냄
- `70`, `75`, `90` → 카메라 촬영 각도
- `000` → 수평 각도 (0°)
- `200` → 크기 설정값

## 7. 촬영 조건 분포

JSON 메타데이터에 포함된 촬영 조건:

| 메타데이터 키 | 설명 |
|-------------|------|
| `camera_la` | 카메라 촬영 각도 (70°, 75°, 90°) |
| `light_color` | 조명 색 (주백색 등) |
| `drug_dir` | 약 촬영 방향 (앞면 / 뒷면) |
| `back_color` | 배경 색 |

> 동일한 알약을 다양한 각도와 조명 조건에서 촬영한 데이터로 구성되어 있어 촬영 환경 변화에도 비교적 안정적인 인식이 가능하다.
