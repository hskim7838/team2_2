TTA 적용 계획

사용할 변형

알약은 방향에 무관한 객체라 flip이 가장 자연스럽습니다.


원본 이미지          → 예측 A

좌우 반전 (H-flip)   → 예측 B → 좌표 복원

상하 반전 (V-flip)   → 예측 C → 좌표 복원

좌표 복원 방법

# H-flip 복원
x1_orig = W - x2_flip
x2_orig = W - x1_flip
y 좌표는 그대로

# V-flip 복원
y1_orig = H - y2_flip
y2_orig = H - y1_flip
x 좌표는 그대로

병합 방법

3개 예측을 WBF로 합칩니다.


A + B + C → WBF (iou_thr=0.5) → 최종 예측

기대 효과
recall 향상: 원본에서 놓친 박스를 flip에서 탐지
위치 정확도 향상: 여러 예측의 평균으로 bbox가 더 정밀해짐

주의사항

추론 속도 3배 느려짐 (test 이미지 수가 적으니 문제 없음)
WBF iou_thr 튜닝 필요 (0.5 시작)

예상 : 소폭 상승하거나 아니면 그대로

이유

향상 가능성 (긍정적)

원본에서 놓친 박스를 flip에서 잡을 수 있음
WBF로 bbox 위치가 더 정밀해짐

제한 요인 (부정적)

이미 threshold=0.1로 거의 모든 탐지를 잡고 있는 상태
알약 이미지가 이미 정면 촬영 → flip해도 큰 차이 없을 수 있음
val grid search에서 0.05~0.1 구간 동일했던 것처럼, 이미 recall이 포화 상태일 수 있음

WBF iou_thr=0.5가 핵심 변수인데, 너무 낮으면 중복 박스가 살아남고 너무 높으면 유효한 박스가 합쳐집니다. 결과를 보고 0.4~0.6으로 조정해볼 수 있습니다.

결과

score : 0.96756

원인 분석

WBF가 오히려 정확한 박스를 흐트러뜨렸을 가능성이 높습니다. flip된 이미지에서 나온 박스와 원본 박스가 완전히 일치하지 않으면 WBF 평균으로 bbox 위치가 살짝 틀어집니다. mAP@0.75:0.95는 위치 정확도에 매우 민감한 지표라 이 영향이 크게 나타납니다.

H-flip만 TTA 시도 (원본 + H-flip WBF)

score : 0.97512

V-flip이 오히려 방해가 됐던 걸로 보입니다. 알약은 상하 방향은 의미가 있지만 (위에서 찍은 사진), 좌우는 대칭이라 H-flip만 자연스러운 augmentation이었던 것 같습니다.

iou_thr의 역할
두 bbox가 "같은 객체"로 판단되는 IoU 기준값입니다.

낮을수록 → 조금만 겹쳐도 하나로 합침 → bbox 수 감소, 위치가 평균화됨
높을수록 → 많이 겹쳐야 하나로 합침 → bbox를 더 독립적으로 유지

0.6 vs 0.7 예상

iou_thr	                               효과	                                                       예상 결과
0.5 (현재)	                    원본+H-flip 적당히 합침	                                         0.97512
0.6	                조금 덜 합침 → 원본 bbox에 더 가까움	                     소폭 상승 가능
0.7	                       거의 안 합침 → TTA 효과 감소	                    TTA 없는 것과 비슷해질 수 있음

알약 탐지 특성상 원본+H-flip 두 예측이 거의 같은 위치에 찍히기 때문에 IoU가 보통 0.8~0.9로 높습니다. 그래서 0.6, 0.7 모두 잘 합쳐질 것 같습니다.

0.6이 더 안전한 선택이고, 0.7은 TTA 효과가 희석될 가능성이 있습니다.
